# OpenShift Builds Operator

## Architecture

This is a kubebuilder-based operator managing two operands: Shipwright Build and Shared Resource CSI Driver.

- `internal/` splits by operand: `shipwright/`, `sharedresource/`, plus `controller/` and `common/`
- `config/` mirrors the split: `config/shipwright/` and `config/sharedresource/` alongside standard kubebuilder dirs (`crd/`, `manager/`, `rbac/`)
- `config/catalog/` and `config/olm/` hold OLM-specific config
- `bundle/` is entirely generated by `make bundle` — never hand-edit
- `test/` has three tiers: `e2e/`, `integration/`, and unit tests alongside source
- `hack/` contains `update-manifests-from-snapshot.sh` — key automation script used by release tooling to update component digests from Konflux snapshots
- API types live in `api/v1alpha1/`

## Build & Test Commands

- Build: `make build` or `go build ./...`
- Test all: `make test`
- Test single package: `go test ./internal/controller/...`
- Lint: `make lint`
- Lint fix: `make lint-fix`
- Format: `make fmt`
- Vet: `make vet`
- Generate manifests: `make manifests`
- Generate code: `make generate`
- Regenerate bundle: `make bundle`

## Key Conventions

- Always run `make bundle` after changing Makefile VERSION, CHANNELS, or Dockerfile labels
- Never edit files under `bundle/` directly — they're generated by `make bundle`
- Never modify `.tekton/` files on main — only on release branches
- `config/manifests/bases/` CSV holds component digests — modified by automation, not manually
- Unit tests run locally; integration/e2e tests require a running OpenShift cluster
- Cross-platform sed: detect Darwin vs Linux for `-i` flag when writing scripts
- Always run `make manifests generate` after changing API types in `api/`

## PR Conventions

- Commit message format: `[BUILD-XXXX] fix(deps): description` — Jira key prefix in brackets, conventional commit style
- Commit flags: always use `-s` (sign-off) and `-S` (GPG sign)
- PR title format: `[BUILD-XXXX] builds-X.Y: scope: description` — includes Jira keys and branch version prefix
- PR body: `## Summary` or `## CVE Fix` header, changes as bullet points, `### Jira Issues` with `Resolves: BUILD-XXXX`, ends with `Co-Authored-By: Claude Code`
- Fork-aware push: detects fork vs upstream for `gh pr create --head` flag
- Default to one commit per PR: amend the existing commit and force push. Update commit message, PR title, and PR body to cover the full diff as if done in one shot — no references to "later added" or "fixed after review". User can override to keep multiple commits if needed.
